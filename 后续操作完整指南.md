## 二、本地环境配置

### 2.1 替换项目文件

用最新下载的 `ai-finance-analyzer.tar.gz` 解压覆盖你本地的项目文件夹。
如果你在 Windows 下无法直接解压 `.tar.gz`，可以用 7-Zip（免费软件）右键解压。

### 2.2 安装项目

```powershell
cd "F:\大学\兴趣\bill manager\ai-finance-analyzer\ai-finance-analyzer"

# 安装项目及所有依赖（包括 pytest、openai、python-dotenv 等）
pip install -e ".[all]"
```

这条命令做了三件事：
- 把项目本身注册为可导入的 Python 包
- 安装 `openai`（DeepSeek API 客户端）
- 安装 `pytest`、`ruff`、`python-dotenv` 等开发工具

以后修改源码后不需要重新安装，改动立即生效（`-e` 是 editable 模式）。

### 2.3 创建 .env 文件

在项目根目录（和 `README.md` 同级）创建 `.env` 文件：

```powershell
# 用 PowerShell 创建 .env 文件
Set-Content -Path ".env" -Value "DEEPSEEK_API_KEY=sk-你的新Key"
```

或者用任何文本编辑器（VS Code、记事本均可）新建文件，内容为：

```
DEEPSEEK_API_KEY=sk-你的新Key
LLM_MODEL=deepseek-chat
DEBUG=false
```

文件名就叫 `.env`（注意前面有个点）。Windows 资源管理器可能不允许创建以点开头的文件名，用 PowerShell 或 VS Code 创建即可。

这个文件已经在 `.gitignore` 中，不会被上传到 GitHub。

## 三、用真实微信账单测试

### 3.1 导出微信账单

1. 打开微信 → 底部「我」→「服务」→「钱包」
2. 右上角「账单」→ 右上角「...」→「导出账单」
3. 选择「用于个人对账」
4. 选择时间范围（建议至少选 3 个月，数据越多分析越准）
5. 微信会在 24 小时内发送 CSV 文件到你绑定的邮箱
6. 从邮箱下载 CSV 文件

### 3.2 运行分析

把下载的 CSV 文件放到项目目录下，然后：

```powershell
python -c "
import sys; sys.path.insert(0, 'src')
from analyzer import run_pipeline

report = run_pipeline('微信支付账单(20260101-20260401).csv')

# 打印关键指标
a = report.analysis
print(f'Period: {a.start_date} to {a.end_date}')
print(f'Transactions: {a.total_transactions}')
print(f'Total Expense: CNY {a.total_expense:,.2f}')
print(f'Total Income:  CNY {a.total_income:,.2f}')
print(f'Savings Rate:  {a.savings_rate:.1%}')
print(f'Health Score:  {a.health_score.total_score:.0f}/100 ({a.health_score.grade})')
print()
print('Category Breakdown:')
for cb in a.category_breakdown:
    print(f'  {cb.category.value:15s} CNY {cb.total:>10,.2f}  ({cb.percentage:.1%})')
print()
print('Risk Alerts:')
for r in a.risk_alerts:
    print(f'  [{r.level.upper()}] {r.message}')
print()
print('AI Advice:')
if report.advice:
    print(f'  {report.advice.summary}')
    for s in report.advice.suggestions:
        print(f'  [{s.priority}] {s.action}')
        print(f'    -> {s.expected_impact}')
"
```

把上面的 CSV 文件名换成你实际下载的文件名。

### 3.3 如果解析失败

微信不同版本导出的 CSV 格式可能有细微差异。如果报错：
- 先用记事本打开 CSV，看一下前 20 行的结构
- 确认是否有「交易时间」「金额(元)」等列名
- 如果格式差异大，可以在 GitHub Issue 中反馈，后续版本会适配

    branches: [main]

## 八、用户视角：使用方式汇总

### 8.1 最简使用：一行代码全流程

```python
from analyzer import run_pipeline

report = run_pipeline("wechat_bill.csv")
print(report.analysis.health_score.total_score)
print(report.advice.summary)
```

### 8.2 不用 AI：纯本地分析

```python
from analyzer import run_pipeline

# use_llm_advice=False 不调用任何外部 API
report = run_pipeline("wechat_bill.csv", use_llm_classify=False, use_llm_advice=False)
```

完全本地运行，零网络请求，零 API 成本。

### 8.3 分步调用：灵活组合

```python
from analyzer.parser import parse_file
from analyzer.classifier import classify
from analyzer.engine import analyze
from analyzer.advisor import generate_advice

# 第一步：解析
transactions = parse_file("bill.csv")

# 第二步：分类（不用 LLM）
categorized = classify(transactions, use_llm_fallback=False)

# 第三步：分析
report = analyze(categorized)

# 到这里就拿到了所有结构化数据，可以自由使用：
print(report.health_score)
print(report.category_breakdown)
print(report.risk_alerts)

# 第四步（可选）：生成 AI 建议
advice = generate_advice(report)
print(advice.suggestions)
```

### 8.4 集成到自己的应用

```python
# 例如：集成到 Flask Web 应用
from flask import Flask, request, jsonify
from analyzer import run_pipeline

app = Flask(__name__)

@app.route("/analyze", methods=["POST"])
def analyze_bill():
    file = request.files["bill"]
    file.save("/tmp/bill.csv")
    report = run_pipeline("/tmp/bill.csv", use_llm_advice=True)
    return jsonify({
        "health_score": report.analysis.health_score.total_score,
        "savings_rate": report.analysis.savings_rate,
        "suggestions": [
            {"action": s.action, "impact": s.expected_impact}
            for s in report.advice.suggestions
        ],
    })
```

### 8.5 自定义分类规则

复制内置规则文件，修改后使用：

```powershell
# 复制规则文件
Copy-Item "src\analyzer\classifier\rules.json" "my_rules.json"
# 用编辑器修改 my_rules.json，添加你自己的关键词
```

```python
from analyzer.classifier import classify

# 使用自定义规则
categorized = classify(transactions, rules_path="my_rules.json")
```

### 8.6 切换 LLM 提供商

默认用 DeepSeek，但可以切换到任何 OpenAI 兼容接口：

`.env` 文件中修改：

```
# 用 OpenAI

LLM_BASE_URL=https://api.openai.com/v1
LLM_MODEL=gpt-4o-mini

# 用本地 Ollama

LLM_BASE_URL=http://localhost:11434/v1

```

- 为什么用四层架构而不是端到端 AI
- 为什么 AI 只用在建议生成而不是分析计算
- 为什么用"规则优先 + AI 兜底"的分类策略
- 健康评分的五维加权算法如何设计

**产品思维方向：**
- 为什么定位为 SDK 而不是 App
- 与 Firefly III 等成熟工具的差异化分析
- 如何定义"决策引擎"与"记账工具"的本质区别

**工程能力方向：**
- 插件化 Parser 架构如何支持格式扩展
- 隐私保护设计（只发送聚合数据给 LLM）
- 测试覆盖率和 CI/CD 实践
